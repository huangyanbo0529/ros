1-27 【项目作业】线程安全日志器
实践描述：
亲爱的慕友们，在我们学习了 C++ 的引用、默认参数和类型安全等基础特性后，是不是感受到它比 C 语言更简洁、更安全？这些看似微小的改进，恰恰是 ROS2 节点 API 设计优雅的关键！现在，让我们通过一个贴近机器人开发场景的小项目，夯实函数设计的基础能力。欢迎小伙伴一起讨论：为什么现代 C++ 接口要避免裸指针和可变参数？
题目：
在多线程 ROS2 节点中，多个回调函数可能同时写日志到 std::cout，导致输出字符交错（如 ThreLogad 1: msg）。请设计一个线程安全的日志类，确保每条日志原子输出，且支持后台线程独立运行。

调用示例与输出：

[LOG] Thread A: message 0
[LOG] Thread B: message 0
[LOG] Thread A: message 1
...
1
2
3
4
代码块
任务描述
一、开发语言与环境要求
1、语言版本：c++语言
2、开发工具：VS Code（需能一键编译运行）

二、程序整体要求

log() 方法必须保证整条消息原子输出
主线程需等待足够时间让后台线程完成（用 sleep 模拟）
不允许使用全局 mutex（必须封装在类内）
三、详细设计规范
1、创建 3 个线程，每个打印 5 条不同消息
2、每条消息包含线程标识（如 “Thread 1”）和序号
3、主线程 sleep 至少 1 秒确保子线程完成

四、提交内容
1、logger.h + main.cpp
2、输出截图或文本（证明无交错）
3、回答：“为何不能用 shared_mutex？”（简答）
